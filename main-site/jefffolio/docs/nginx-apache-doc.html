Nginx & Apache Documentation

Documentation covering the deployment and configuration of the Nginx reverse proxy and Apache web server.

Overview

Nginx is used as the public-facing reverse proxy while Apache hosts the application content internally.

Purpose

  • Centralized SSL termination
  • Reverse proxy routing
  • Host multiple services
  • Hide internal containers
  • Improve security posture

Architecture

Internet
    |
Nginx Reverse Proxy
    |
Apache Container

Installation Steps

  1. Install Docker and Docker Compose
  2. Create Docker network
  3. Deploy Apache container
  4. Deploy Nginx container
  5. Configure reverse proxy rules
  6. Configure SSL certificates

Configuration Examples

upstream apache_backend {
    server apache:80;
}

location / {
    proxy_pass http://apache_backend;
}
